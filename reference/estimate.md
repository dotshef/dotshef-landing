너는 시니어 프론트엔드 엔지니어다. Next.js(App Router)로 “견적 자동 계산” 정적 웹페이지를 구현하라.
목표는 사용자가 단계별 질문(Flow)으로 선택을 하면, 프로젝트 타입별 base + 옵션 가격을 합산해 예상 견적을 보여주는 것이다.
사이트는 정적(SSG)으로 빌드되어야 하며 서버/DB 없이 동작해야 한다. (로컬 상태로만 계산)

[기술 스택/제약]
- Next.js 14+ App Router
- TypeScript
- 정적 웹사이트(SSG): 서버 API 호출 없음
- UI는 TailwindCSS 사용 (기본적인 스타일만, 과하지 않게 깔끔하게)
- 상태 관리는 React useState/useMemo 정도면 충분
- 데이터(가격표)는 코드 내 상수(JSON 객체)로 정의 (향후 엑셀/JSON으로 교체 가능하도록 구조화)
- 결과를 “총합”으로 보여주고, 각 항목별 비용 breakdown(베이스/추가 기능)을 리스트로 보여줄 것

[가격 정책(현재 버전)]
프로젝트 타입 3개 + ‘여기에 없어요’ 분기.

1) 랜딩 페이지(마케팅용)
- 설명: 외부 링크, 반응형, SEO 기본
- base: 200,000원
- 추가 기능(체크박스)
    - 문의 입력 폼(이메일): +200,000원
    - 지도: +200,000원
- 주의: 랜딩에서는 게시물/권한 질문은 나오면 안 됨 (분기 필수)

2) 콘텐츠형(CMS)
- 설명: 게시물 추가/수정/삭제, 로그인, 관리자 기능
- base: 600,000원
- 추가 질문/기능
    - 문의 입력 폼(이메일): +200,000원
    - 지도: +200,000원
    A. 게시물 종류 개수 (숫자 입력, 최소 1)
      - 종류당 +200,000원
      - 여기서 “게시물 종류”는 게시판/콘텐츠 타입을 의미
    B. 게시물에 이미지 등 파일 첨부 가능 여부 (Yes/No)
      - Yes일 경우 +200,000원
      - 조건: 게시물 종류가 1개라도 첨부 가능하면 비용 추가됨

3) 데이터관리 SaaS
- 설명: 로그인 + 권한 별 액션 분리, 데이터 취합
- base: 800,000원
- 추가 질문/기능
  - 문의 입력 폼(이메일): +200,000원
  - 지도: +200,000원
  A. 데이터/게시물 종류 개수 (숫자 입력, 최소 1)
    - 종류당 +200,000원
      B. 필요한 권한 종류 개수 (숫자 입력, 최소 1)
    - 종류당 +100,000원

4) “여기에 없어요”
- 선택하면 계산기 대신 안내 문구를 보여주고, “문의하기” 버튼/링크(예: mailto 또는 /contact 섹션 앵커)로 이동시켜라.
- 문구 예: “요구사항이 복잡하거나 위 항목에 없다면, 간단히 텍스트로 설명해 주세요. 확인 후 맞춤 견적을 드립니다.”

[UI/Flow 요구사항]
- 1단계: 프로젝트 타입 선택 카드(3개 + ‘여기에 없어요’)
- 2단계: 타입에 맞는 추가 질문을 단계적으로 노출
    - 랜딩: 체크박스 2개(문의폼, 지도)
    - CMS: 숫자 입력(게시물 종류) + 토글(파일 첨부)
    - 데이터관리 SaaS: 숫자 입력(종류 수) + 숫자 입력(권한 종류 수)
- 3단계: 결과 패널
    - 총 견적(원 단위 포맷팅)
    - 소비자가 확정하지 못하도록 가격은 범위별로 알려주도록 할 것.
- 사용자가 타입을 변경하면, 이전 타입에 해당하는 입력값은 초기화되어야 함

[구현 구조]
- /app/page.tsx 하나로 구현해도 좋고, components 폴더로 분리해도 좋다.
- 가격/질문 정의는 다음처럼 “데이터 기반”으로 확장 가능하도록 설계:
    - ProjectType 정의 (id, title, description, basePrice)
    - Question 정의 (id, type: checkbox|number|toggle, visibility rule by projectType)
    - PricingRule 함수(answers -> lineItems[])로 계산
- lineItems[]는 { label, amount } 형태로 만들고 합산해서 total을 출력하라.

[추가]
- 입력값 검증: 숫자 입력은 1 이상만 허용. 0 이하 입력 시 자동으로 1로 보정하거나 에러 문구 표시.
- 금액 포맷: Intl.NumberFormat('ko-KR') 사용.

[산출물]
- 완성된 코드 전체(필요 파일들) 제시
- 실행 방법(install, dev, build) **간단히 안내**
