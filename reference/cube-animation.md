# 🧊 3×3 큐브 블록 3D 애니메이션 기획서 (v3.0)

> **프로젝트**: 포트폴리오 랜딩페이지 히어로 섹션 — 우측 3D 애니메이션 컴포넌트  
> **작성일**: 2026년 3월 17일  
> **버전**: v3.0  
> **다크모드**: 미지원 (라이트 모드 전용)

---

## 1. 개요

### 1.1 목적
"당신을 위한 맛있는 소프트웨어" 문구 우측에 위치하는 **독립 컴포넌트**로서, 큐브 아래에 깔린 블록 평면에서 블록이 하나씩 추출되어 포물선 궤적으로 날아가 3×3 큐브로 조립되는 3D 애니메이션을 구현한다.

### 1.2 핵심 컨셉
- **"혼돈에서 질서로"** — 평평하게 흩어진 블록들이 하나의 입체적 구조물로 조립됨
- 블록이 무작위 순서로 추출되어 예측 불가능한 리듬감 부여
- 평면에서 블록이 빠지면 아래에서 새 블록이 올라와 채워지는 **자급자족 순환 시스템**

### 3.2 카메라 설정
- 종류: PerspectiveCamera
- 시점: 정면에서 약간 위(약 20~25도 하향)로 비스듬히 내려다보는 각도
- 큐브 조립 영역과 아래 평면이 모두 자연스럽게 시야에 들어오도록 구성
- FOV: 45~50도

## 4. 블록 평면 상세

### 4.1 평면 스펙

```
그리드: 5열 × 6행 = 총 30칸
필요 블록: 27개 (큐브 조립용)
여유 칸: 3개 (빈 자리 보충용 버퍼)
블록 크기: 큐비와 동일 (1 × 1 × 1 단위, bevel 0.05)
블록 간 간격: 0.08 단위
평면 위치: 큐브 중심에서 Y축 아래로 약 5~6 단위
```

### 4.2 평면 블록 색상
- 기본 상태: Secondary (`#F5F5F5`) — 밝은 회색으로 평면에 안착
- 추출 선택 순간: Accent (`#4A90D9`) 하이라이트가 0.2s간 깜빡임
- 추출 후 이동 중: Primary (`#1A1A1A`)로 색상 전환 — 큐브 블록으로 변신

### 4.3 빈 자리 보충 메커니즘

```
1. 블록이 평면에서 위로 솟아올라 추출됨 (Y축 +0.5 → 이후 포물선 이동)
2. 해당 칸이 빈 자리가 됨 (0.3s간 빈 상태 유지)
3. 아래(Y축 -)에서 새 블록이 올라오며 빈 자리를 채움
   - 이징: "back.out" (살짝 튀어오르며 착지)
   - 소요 시간: 0.4s
4. 평면이 다시 평평한 상태로 복귀

시각적 효과:
- 빈 자리에 미세한 그림자 홀(shadow hole) 표현 → 아래에서 올라오는 블록이 자연스럽게 보임
- 보충 블록은 Secondary 색상으로 올라옴
```

## 5. 애니메이션 시나리오

### 5.1 전체 타임라인

| 단계 | 시간 | 설명 |
|------|------|------|
| **Phase 1 — 장면 등장** | 0.0s ~ 1.0s | 평면이 먼저 페이드인 + 스케일업 등장. 이후 큐브 조립 영역(빈 와이어프레임 or 투명)이 미세하게 표시 |
| **Phase 2 — 블록 추출 & 조립** | 1.0s ~ 8.0s | 27개 블록이 하나씩 평면에서 추출 → 포물선 이동 → 큐브에 안착. 블록당 약 0.25s 간격 |
| **Phase 3 — 완성 피드백** | 8.0s ~ 8.8s | 마지막 블록 안착 후 Accent 글로우 스캔 + 큐브 미세 바운스 (1.0 → 1.03 → 1.0) |
| **Phase 4 — 평면 퇴장** | 8.8s ~ 9.5s | 평면이 아래로 서서히 내려가며 페이드아웃. 큐브만 남음 |
| **Phase 5 — 아이들** | 9.5s~ | 완성된 큐브가 Y축 기준 천천히 자동 회전 (8초/1회전). 루프 |

### 5.2 블록 추출 & 이동 시퀀스 (1블록당)

```
[0.00s] 평면에서 해당 블록 Accent 하이라이트 깜빡임
[0.15s] 블록이 평면에서 Y축으로 솟아오름 (+0.5 단위, 0.2s)
[0.20s] 색상 전환: Secondary → Primary (페이드 0.15s)
[0.35s] 솟아오른 블록이 포물선 궤적으로 큐브 목표 위치를 향해 이동 시작
[0.80s] 큐브 목표 위치에 도착, 미세 바운스로 착지 (0.1s)
[0.90s] 해당 블록 조립 완료

동시에 평면에서는:
[0.35s] 빈 자리 아래에서 보충 블록이 올라오기 시작
[0.75s] 보충 블록 착지, 평면 복구 완료
```

### 5.3 블록 추출 순서 (무작위 느낌)

```
큐브 블록 넘버링 (3×3×3):

      Layer 1 (상단)     Layer 2 (중간)     Layer 3 (하단)
      ┌───┬───┬───┐     ┌───┬───┬───┐     ┌───┬───┬───┐
      │ 1 │ 2 │ 3 │     │10 │11 │12 │     │19 │20 │21 │
      ├───┼───┼───┤     ├───┼───┼───┤     ├───┼───┼───┤
      │ 4 │ 5 │ 6 │     │13 │14 │15 │     │22 │23 │24 │
      ├───┼───┼───┤     ├───┼───┼───┤     ├───┼───┼───┤
      │ 7 │ 8 │ 9 │     │16 │17 │18 │     │25 │26 │27 │
      └───┴───┴───┘     └───┴───┴───┘     └───┴───┴───┘

추출 순서 (무작위 느낌 + 구조적 안정성 확보):
5 → 4 → 6 → 7 → 9 → 10 → 1 → 2 → 3 →
15 → 12 → 8 → 14 → 11 → 13 → 16 → 18 → 17 →
27 → 19 → 25 → 20 → 21 → 22 → 24 → 26 → 23

설계 의도:
- 중간층·상단층 블록이 먼저 배치되어 "공중에 뜨는" 신비감
- 하단층이 나중에 채워지며 구조가 완성되는 만족감
- 인접 블록이 연속되지 않도록 배치하여 무작위 느낌 극대화
```

### 5.4 포물선 궤적 디테일

```
궤적: 3D 베지어 곡선 (Cubic Bezier)

시작점: 평면에서 솟아오른 위치 (평면 Y + 0.5)
정점:   시작점과 목표점 사이 중간 지점에서 Y축으로 +2.0~3.0 단위 위
끝점:   큐브 내 목표 슬롯 위치

각 블록마다 정점 높이에 ±0.5 랜덤 편차를 줘서
모든 블록이 동일한 궤적을 그리지 않도록 함

이동 시간: 0.45s
이징: "power2.inOut" (부드러운 출발 → 부드러운 착지)
이동 중 블록 회전: Y축 기준 180~360도 회전 (비행감)
```

### 5.5 루프 정책
- 최초 방문 시: Phase 1~5 전체 재생 (약 10초)
- 이후: Phase 5(아이들 회전)만 무한 루프
- 뷰포트 이탈 시: `Intersection Observer`로 애니메이션 일시 정지

## 6. 3D 큐브 & 큐비 스펙

### 6.1 큐브 구조

```
구조: 3 × 3 × 3 = 총 27개 큐비
큐비 크기: 1 × 1 × 1 단위
큐비 간 간격: 0.08 단위
모서리 라운딩: bevel 0.05 (둥근 사각형)
전체 큐브 크기: 약 3.24 × 3.24 × 3.24 단위
```

### 6.2 재질 & 라이팅

**재질**
- 큐비 (조립 후): Primary `#1A1A1A` — MeshStandardMaterial
    - `roughness`: 0.7 (무광)
    - `metalness`: 0.0
- 평면 블록: Secondary `#F5F5F5` — MeshStandardMaterial
    - `roughness`: 0.8
    - `metalness`: 0.0

**라이팅**
- Ambient Light: 강도 0.5, `#FFFFFF`
- Directional Light: 우상단 45도 (강도 0.8)
- 소프트 섀도우: 큐브 아래 + 평면 위에 은은한 그림자

**후처리**
- Bloom: 없음
- SSAO: 약하게 (큐비 틈새 깊이감)
- 캔버스 배경: 투명

## 10. 기술 스택

### 10.1 권장 스택

```
3D 렌더링      : Three.js (r160+) 또는 React Three Fiber
애니메이션      : GSAP Timeline + CustomEase (포물선 경로)
후처리          : @react-three/postprocessing (SSAO)
경로 계산       : Three.js CubicBezierCurve3 (포물선 궤적)
```

### 10.2 핵심 구현 구조

```
Scene
├── CameraRig (마우스 패럴랙스)
├── Lights (Ambient + Directional)
├── CubeAssembly (큐브 조립 영역)
│   └── CubiSlot × 27 (각 큐비 목표 위치)
├── BlockPlane (5×6 그리드 평면)
│   ├── PlaneBlock × 30 (평면 블록들)
│   └── RefillSystem (보충 블록 풀)
├── FlyingBlock (현재 이동 중인 블록, 최대 1개)
└── PostProcessing (SSAO)
```
